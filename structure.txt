---------- SETUP ----------

> .next
    This is where Next.js stores the machine code after compiling it.
    We do not need to touch this.
    However if this is deleted, it will just be regenerated the next time we run the website.

> node_modules
    This is where Next.js stores external libraries like Tailwind or Next.js itself.
    We do not need to touch this.
    However if this is deleted or edited, it will just be overwritten the next time we run the website.

> .venv
    (If you don't have a virtual environment I would suggest getting it)
    This is your Python virtual environment.
    This is the projects own version of Python.
    We use this to separate the global Python version from the project version.
    After creating this you do not need to touch it.

---------- CONFIGURATION ----------

package.json
    The project 'recipe'.
    This lists libraries we need and commands we run (like 'npm run dev').
    You edit this to add new tools.

package-lock.json
    The 'exact version'-log. It ensures everyone has an identical setup.
    DO NOT edit this manually, it updates automatically when you install things.


next.config.ts
    This contains settings for Next.js.

tsconfig.json & next-env.d.ts
    These are for TypeScript. they make sure that our code is type-safe.
    We do not have to touch them, but we can use them to create aliases like:
        @/components instead of ../../components.

postcss.config.mjs & eslint.config.mjs
    These are for Tailwind and code-quality checking.
    We do not really have to touch them.

.env
    This contains environment variables like API-keys.
    DO NOT upload this to GitHub.
    Make sure that it is part of .gitignore.

.gitignore
    This is where we tell Git which files to NOT upload to GitHub.
